Ask only questions that are absolutely neccessary for ulitmate user experience

  1. It's a good idea to provide user with the steps ahead
  2. If password required, please make sure you explain if special characters are needed
  3. Error codes need to be better explained instead of "something went wrong" or "unable to log in"
  4. Confirmation required so that users knows whether the form has been submitted
  5. Explain if the payament is required beforehand
  6. Make sure the form is in logical order (people don't really read a lot)
  7. If form is really long, make sure to break it in few steps / different pages with breadcrumps
  8. Always think twice if you need that information from the user

Data Privacy

Remember that Many countries have laws about when and how you can collect personal information, how to securely store that data, and what you can do with the personal information.

Placeholders

Don't put a text in every placeholder just because you can, it normally confuses users, and they skip it and don't read it, what's worse might type in the exact text that's written in placeholder. Use it to guide user if the item to type is very specific

Common attributes used in forms

  1. input type = "button / checkbox / color / date / email / file / hidden / image / month / search / tel / etc."
  2. input name = labels the data when it's sent to the server or database, or however the data is processed, and you can think of it like the column name in the database. Each name in a form should be unique. This isn't shown to the user.
  3. input for = links label with ID, make sure you include in
  4. input value= For fields where a user enters data such as text, number, or date, the value is the data that the user fills into the form field. You can also use the value attribute to set a default value of a field when the form is first loaded. The user will see this as data that's already filled in but they can change it by entering their own data. The next is in a form field where the user chooses from options you supply to them, such as checkboxes, radio buttons, or select menu. So there should be a value for each checkbox, or a value for each option in a select element. The value attribute is also used on button-type inputs to set the text that is displayed on the button.
  5. input= checked / selected
  6. input = placeholder (helps with the answer)
  7. input = min / max
  8. input = step
  9. input = maxlength="value" / max number of characters allowed
  10. multiple value / allows enter multiple values eg. input type="email" multiple
  11. input type="text" PATTERN="[A-Za-z] {3}"
  12. input type="text" REQUIRED (must be filled in)
  13. input type="text" ARIA-REQUIRED="true" (aria will tell that this is required )
  14. input type="text" DISABLED (either to one or multiple fields)
  15. input type="text" readonly (either to one or multiple fields)
  16. input type="text" HIDDEN (user won't see it, good for tracking)
